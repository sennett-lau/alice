## What changed

The wiki load policy flipped from "auto-load `docs/wiki/**`" to "auto-load `docs/wiki/README.md` (index) + `docs/wiki/current-status.md` (dynamic state) only — every other wiki page loads on demand." As wikis grow page-by-page, the old policy quietly burned context every session for content the agent didn't need. The new policy keeps the always-on surface tiny and pushes the rest behind an index.

To make on-demand loading actually work, `docs/wiki/README.md` is now a strict index: each entry follows the contract `path — what's inside, when to query it`. Without the "when to query" half the agent either reads everything or reads nothing, so both halves are load-bearing. The `wiki-maintainer` sub-agent now owns the index alongside the pages.

A small-wiki escape hatch: if your wiki has fewer than 4 content pages beyond `README.md` and `current-status.md`, auto-loading the whole directory is still acceptable — the indirection isn't worth it at that size. Flip to index-only as soon as you cross the threshold.

The change touches three places adopters depend on: the binding rule (`framework/rules/docs-layout-and-load-policy.md`), the adopter's `CLAUDE.md` load-policy table (seeded from `template/CLAUDE.md`), and `docs/README.md` + `docs/wiki/README.md` (seeded from `template/docs/`). `/sync` updates the framework rule automatically; the others are owned by the adopter, so they appear in manual actions below.

This release also adds a new **Principles** section to `.alice/rules/implementation-quality.md` — eight principles (Simplicity, Clarity, Flexibility at proven seams, Modularity, Consistency, Efficiency, Resilience, Refactoring within blast radius) that sit above the existing non-negotiables. No structural migration needed; `/sync` copies the updated rule file in place. If your project's `CLAUDE.md` quotes or paraphrases the old `implementation-quality.md` text, re-skim it against the new content for drift.

## Automatic actions

None. The load-policy rule under `.alice/rules/` is updated by the standard `/sync` file copy. Adopter-owned files (`CLAUDE.md`, `docs/README.md`, `docs/wiki/README.md`) are not touched automatically — see manual actions.

## Manual actions

- [ ] Update `CLAUDE.md` load-policy table: replace the `docs/wiki/**` row with three rows — `docs/wiki/README.md` (auto-load), `docs/wiki/current-status.md` (auto-load), `docs/wiki/<page>.md` other (load on demand). See `.alice/_alice/template/CLAUDE.md` for the canonical wording.
- [ ] Update `docs/README.md` load-policy table the same way. Reference: `.alice/_alice/template/docs/README.md`.
- [ ] Audit `docs/wiki/README.md`. Every `*.md` file in `docs/wiki/` must have one index line following `path — what's inside, when to query it`. Both halves required; the "when to query" half is what lets agents pull pages on demand. Vague triggers like "for reference" defeat the policy — make them concrete.
- [ ] If your wiki has fewer than 4 content pages beyond `README.md` and `current-status.md`, you can keep `docs/wiki/**` auto-loaded for now and revisit when the wiki grows. Note the choice in `CLAUDE.md` so the next agent doesn't second-guess it.
- [ ] Run a quick lint after the audit: `wiki-maintainer` in lint mode now flags missing index entries, deleted-page references, and vague triggers — invoke it once to validate the new index.
- [ ] Re-read `.alice/rules/implementation-quality.md` after the sync — the new **Principles** section may shift how you frame ambiguous calls. If your `CLAUDE.md` paraphrased or contradicted the old version, reconcile.
